## B — Ikki tillilik (i18n: EN / UZ)

### 1. Til fayllari
Laravel `lang/` tarjima papkalari:
```
lang/
├── en/
│   ├── common.php       # umumiy: save, cancel, delete, search, actions...
│   ├── nav.php          # sidebar: dashboard, leads, campaigns, analytics...
│   ├── leads.php        # mijozlar moduli matnlari
│   ├── campaigns.php    # kampaniyalar moduli matnlari
│   ├── analytics.php    # tahlil moduli matnlari
│   ├── auth.php         # login matnlari
│   └── validation.php   # validatsiya xabarlari (EN)
└── uz/
    ├── common.php
    ├── nav.php
    ├── leads.php
    ├── campaigns.php
    ├── analytics.php
    ├── auth.php
    └── validation.php
```
Barcha blade'dagi qattiq yozilgan matnlar `{{ __('leads.title') }}` ko'rinishiga
o'tkaziladi.

### 2. Til almashtirish mexanizmi
- **`SetLocale` middleware** — `session('locale')` yoki cookie'dan tilni o'qiydi,
  `app()->setLocale()` ni o'rnatadi. `bootstrap/app.php` web guruhiga qo'shiladi.
- **`LocaleController@switch`** — `GET /locale/{locale}` (en|uz), sessiyaga yozadi,
  orqaga qaytaradi. Route: `locale.switch`.
- **Header'da til tanlash** — bayroq/ikonka bilan EN ⇄ UZ dropdown (dashboard
  layout'ning header qismiga).
- Standart til: `uz` (`.env` `APP_LOCALE=uz`, `APP_FALLBACK_LOCALE=en`).

### 3. Tarjima qilinadigan joylar
- Sidebar bo'limlari va linklar (`nav.php`)
- Sahifa sarlavhalari, breadcrumb, header
- Jadval ustunlari, tugmalar, bo'sh holat matnlari
- Forma label'lari, placeholder'lar, yordamchi matnlar
- Toast turlari (Muvaffaqiyat/Xatolik/Ogohlantirish/Ma'lumot)
- Modal matnlari (o'chirish, logout)
- Status/source/channel yorliqlari (enum → tarjima xaritasi)
- Controller'lardagi `with('success', ...)` xabarlari → `__('...')`
- Validatsiya xabarlari → `validation.php` + `attributes`

### 4. Enum yorliqlarini tarjima qilish
`status`, `source`, `channel` qiymatlari DB'da inglizcha kalit (`new`, `email`...)
bo'lib qoladi, faqat **ko'rsatishda** tarjima qilinadi:
```php
__('leads.status.' . $lead->status)   // "Yangi" yoki "New"
```

## C — Seederlar (barcha jadvallar uchun)

### Factory'lar
- `CampaignFactory` — tasodifiy nom, kanal, holat, byudjet/sarflangan, sanalar
- `LeadFactory` — tasodifiy mijoz, manba, holat, qiymat, kampaniya/xodimga bog'lash

### Seederlar
| Seeder | Tavsif |
|--------|--------|
| `RolePermissionSeeder` | (mavjud) ruxsatlar + superadmin/manager |
| `UserSeeder` | superadmin + bir nechta manager/xodim demo foydalanuvchi |
| `CampaignSeeder` | ~12 ta realistik kampaniya (turli kanal/holat) |
| `LeadSeeder` | ~60 ta mijoz, kampaniyalarga va xodimlarga taqsimlangan |

`DatabaseSeeder` chaqiruv tartibi:
`RolePermissionSeeder → UserSeeder → CampaignSeeder → LeadSeeder`
(FK bog'liqliklar to'g'ri tartibda).

Natija: `php artisan migrate:fresh --seed` dan keyin Tahlil sahifasi **real
grafiklar va raqamlar** bilan to'ladi (bo'sh emas).
